AI Collaboration & Architectural Thought Process
Throughout this project, I used AI (Gemini/Claude) not just as a code generator, but as a product manager, architectural sounding board, and pair-programmer. My goal was to ensure focus remained on structural integrity and product value.

1. Product Discovery & Requirement Analysis
Before writing a single line of code, I sat down with Gemini to deeply analyze the product concept. We brainstormed the core pain points of the target audience (crypto investors), their needs, and the optimal user journey. Only after understanding the "Why" (what makes a personalized crypto dashboard valuable) did I move on to the "How" (technical implementation).

2. Tech Stack Debate & Architectural Design
We had an in-depth debate regarding the tech stack, comparing various frameworks. I actively chose a decoupled architecture using React for the client and FastAPI (Python) for the backend, explicitly rejecting monolithic full-stack frameworks (like Next.js) for this specific use case. My reasoning was twofold:

Academic Principles: I wanted to strongly apply concepts from my academic courses, specifically Object-Oriented Programming (OOP) and SOLID principles. Separating the backend and frontend enforces strict encapsulation, modularity, and a clear separation of concerns.

ML Future-Proofing: Since the core concept revolves around AI curation and a future Machine Learning recommendation pipeline (see Bonus section), writing the backend in Python from day one provides a native ecosystem for ML libraries (like Pandas, Scikit-Learn, or PyTorch).

3. Data Modeling & Resilient Troubleshooting
I leveraged AI to debug specific integration edge cases and brainstorm database designs. For instance, when planning how to store user feedback for future ML training, I wanted a solution that was both structured and flexible. Instead of rigid schema migrations, we discussed ORM behavior and implemented a clean approach: utilizing the existing preferences JSONB column in the User model to store dynamic profiling data (e.g., liked meme categories). This avoids database clutter while perfectly setting up the data structures needed to build user preference vectors for collaborative filtering later.

4. UI/UX & Clean Code Execution
I used AI to help refactor my React components and strip away messy CSS. I guided the AI to implement a modern, clean "Soft UI" (glassmorphism, subtle shadows, fixed-aspect ratios) to prevent layout shifts. I strongly believe that clean code isn't just a backend requirement—the frontend needs to be highly readable, componentized, and robust.
